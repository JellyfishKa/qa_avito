# Баги API

Найдены при прогоне автотестов. Окружение: `https://qa-internship.avito.com`

---

### BUG-001: API принимает отрицательную цену при создании объявления

- **Серьёзность:** Critical
- **Связанный тест:** `test_create_with_negative_price` (TC-004)
- **Шаги воспроизведения:**
  1. Отправить `POST /api/1/item` с телом: `{"sellerId": 555123, "name": "Test", "price": -100, "statistics": {"likes": 1, "viewCount": 1, "contacts": 1}}`
- **Фактический результат:** HTTP 200, объявление создаётся как будто всё нормально
- **Ожидаемый результат:** HTTP 400 — цена не может быть отрицательной

---

### BUG-002: API принимает отрицательные значения в поле statistics

- **Серьёзность:** Major
- **Шаги воспроизведения:**
  1. Создать объявление с корректными данными, убедиться что это работает
  2. Отправить `POST /api/1/item` с телом: `{"sellerId": 555123, "name": "Test", "price": 100, "statistics": {"likes": -1, "viewCount": -5, "contacts": -10}}`
- **Фактический результат:** HTTP 200, объявление создаётся с отрицательными значениями статистики
- **Ожидаемый результат:** HTTP 400 — счётчики не могут быть отрицательными

---

### BUG-003: API отклоняет создание объявления с нулевой статистикой

- **Серьёзность:** Minor
- **Связанный тест:** `test_get_statistic_with_zero_values` (TC-028)
- **Шаги воспроизведения:**
  1. Отправить `POST /api/1/item` с телом: `{"sellerId": 555123, "name": "Test", "price": 100, "statistics": {"likes": 0, "viewCount": 0, "contacts": 0}}`
- **Фактический результат:** HTTP 400
- **Ожидаемый результат:** HTTP 200 — нули это валидное начальное состояние для нового объявления (возможно задумано как защита от нулей, но тогда нужна документация)

### BUG-004: GET /api/1/:sellerID/item возвращает 200 для отрицательного sellerID

- **Серьёзность:** Minor
- **Связанный тест:** `test_get_items_negative_seller_id` (TC-025)
- **Шаги воспроизведения:**
  1. Отправить `GET /api/1/-1/item`
- **Фактический результат:** HTTP 200, пустой массив `[]`
- **Ожидаемый результат:** HTTP 400 или хотя бы 404 — отрицательный sellerID не должен обрабатываться как валидный запрос
