## POST /api/1/item — Создание объявления

### TC-001: Создание объявления с валидными данными
- **Тип:** Positive
- **Приоритет:** High
- **Шаги:**
  1. Отправить POST `/api/1/item` с телом: `name`, `price` (целое > 0), `sellerId` (111111–999999), `statistics` (likes, viewCount, contacts ≥ 0)
- **Ожидаемый результат:** HTTP 200, в ответе присутствует `id` созданного объявления; значения `name`, `price`, `sellerId` совпадают с отправленными

### TC-002: Созданное объявление получает уникальный id
- **Тип:** Positive
- **Приоритет:** High
- **Шаги:**
  1. Создать два объявления с одинаковыми данными
- **Ожидаемый результат:** id у обоих объявлений разные

### TC-003: Создание объявления с нулевой ценой
- **Тип:** Corner case
- **Приоритет:** Medium
- **Шаги:**
  1. Отправить POST с `price: 0`
- **Ожидаемый результат:** HTTP 200 — нулевая цена технически допустима, объявление создаётся; или HTTP 400 с описанием ошибки валидации

### TC-004: Создание с отрицательной ценой
- **Тип:** Negative
- **Приоритет:** Medium
- **Шаги:**
  1. Отправить POST с `price: -100`
- **Ожидаемый результат:** HTTP 400 — отрицательная цена нарушает бизнес-логику

### TC-005: Создание без обязательного поля name
- **Тип:** Negative
- **Приоритет:** High
- **Шаги:**
  1. Отправить POST без поля `name`
- **Ожидаемый результат:** HTTP 400, в теле ответа указано, что поле `name` обязательно

### TC-006: Создание без обязательного поля price
- **Тип:** Negative
- **Приоритет:** High
- **Шаги:**
  1. Отправить POST без поля `price`
- **Ожидаемый результат:** HTTP 400, в теле ответа указано, что поле `price` обязательно

### TC-007: Создание без обязательного поля sellerId
- **Тип:** Negative
- **Приоритет:** High
- **Шаги:**
  1. Отправить POST без поля `sellerId`
- **Ожидаемый результат:** HTTP 400, в теле ответа указано, что поле `sellerId` обязательно

### TC-008: Создание с пустым телом запроса
- **Тип:** Negative
- **Приоритет:** Medium
- **Шаги:**
  1. Отправить POST с пустым телом `{}`
- **Ожидаемый результат:** HTTP 400 — все обязательные поля отсутствуют, должна быть ошибка

### TC-009: Передача строки вместо числа в поле price
- **Тип:** Negative
- **Приоритет:** Medium
- **Шаги:**
  1. Отправить POST с `price: "дорого"`
- **Ожидаемый результат:** HTTP 400 — неверный тип данных

### TC-010: Передача строки вместо числа в поле sellerId
- **Тип:** Negative
- **Приоритет:** Medium
- **Шаги:**
  1. Отправить POST с `sellerId: "abc"`
- **Ожидаемый результат:** HTTP 400 — неверный тип данных

### TC-011: Создание с очень длинным именем (граничное значение)
- **Тип:** Corner case
- **Приоритет:** Low
- **Шаги:**
  1. Отправить POST с `name` длиной 1000 символов
- **Ожидаемый результат:** HTTP 200 — имя сохраняется полностью; или HTTP 400, если на поле есть ограничение по длине

### TC-012: Создание с отрицательными значениями в statistics
- **Тип:** Negative
- **Приоритет:** Medium
- **Шаги:**
  1. Отправить POST с `statistics: {"likes": -1, "viewCount": -5, "contacts": -10}`
- **Ожидаемый результат:** HTTP 400 — счётчики статистики не могут быть отрицательными

### TC-013: Идемпотентность — повторное создание с теми же данными
- **Тип:** Corner case
- **Приоритет:** Medium
- **Шаги:**
  1. Отправить POST дважды с одинаковыми `name`, `price`, `sellerId`
- **Ожидаемый результат:** Оба запроса возвращают HTTP 200 с разными `id` — метод не идемпотентен

### TC-014: Создание со спецсимволами в name
- **Тип:** Corner case
- **Приоритет:** Low
- **Шаги:**
  1. Отправить POST с `name: "<script>alert(1)</script>"`
- **Ожидаемый результат:** HTTP 200, значение сохраняется как есть (экранирование — на стороне клиента при отображении)

## GET /api/1/item/:id — Получение объявления по ID

### TC-015: Получение существующего объявления
- **Тип:** Positive
- **Приоритет:** High
- **Предусловия:** Существует объявление с известным `id`
- **Шаги:**
  1. Создать объявление, сохранить его `id`
  2. Отправить GET `/api/1/item/{id}`
- **Ожидаемый результат:** HTTP 200, в ответе поля `id`, `sellerId`, `name`, `price`, `statistics`, `createdAt` совпадают с переданными при создании

### TC-016: Все поля в ответе присутствуют и имеют правильный тип
- **Тип:** Positive
- **Приоритет:** High
- **Шаги:**
  1. Создать объявление, получить по `id`
  2. Проверить типы каждого поля
- **Ожидаемый результат:** `id` — string, `sellerId` — int, `name` — string, `price` — int, `statistics` — объект, `createdAt` — string

### TC-017: Получение объявления по несуществующему ID
- **Тип:** Negative
- **Приоритет:** High
- **Шаги:**
  1. Отправить GET с `id: "00000000-0000-0000-0000-000000000000"`
- **Ожидаемый результат:** HTTP 404 — объявление не найдено

### TC-018: Передача невалидного формата ID (строка без UUID-формата)
- **Тип:** Negative
- **Приоритет:** Medium
- **Шаги:**
  1. Отправить GET с `id: "abc123"`
- **Ожидаемый результат:** HTTP 400 или HTTP 404 — ID не соответствует ожидаемому формату

### TC-019: Передача пустой строки в качестве ID
- **Тип:** Negative
- **Приоритет:** Low
- **Шаги:**
  1. Отправить GET `/api/1/item/`
- **Ожидаемый результат:** HTTP 400 или HTTP 404 (сервер не должен вернуть 500)

## GET /api/1/:sellerID/item — Получение всех объявлений продавца

### TC-020: Получение объявлений продавца с одним объявлением
- **Тип:** Positive
- **Приоритет:** High
- **Шаги:**
  1. Создать объявление с уникальным `sellerId`
  2. Отправить GET `/api/1/{sellerId}/item`
- **Ожидаемый результат:** HTTP 200, массив содержит ровно одно объявление с нужным `sellerId`

### TC-021: Получение объявлений продавца с несколькими объявлениями
- **Тип:** Positive
- **Приоритет:** High
- **Шаги:**
  1. Создать 3 объявления с одним `sellerId`
  2. Отправить GET `/api/1/{sellerId}/item`
- **Ожидаемый результат:** HTTP 200, массив содержит ровно 3 объявления, у каждого `sellerId` совпадает

### TC-022: Все объявления в ответе принадлежат запрошенному продавцу
- **Тип:** Positive
- **Приоритет:** High
- **Шаги:**
  1. Создать объявления для двух разных `sellerId`
  2. Запросить объявления одного из них
- **Ожидаемый результат:** В ответе нет объявлений другого продавца

### TC-023: Получение объявлений для продавца без объявлений
- **Тип:** Corner case
- **Приоритет:** Medium
- **Шаги:**
  1. Отправить GET с `sellerId`, у которого нет объявлений
- **Ожидаемый результат:** HTTP 200, пустой массив `[]`

### TC-024: Передача невалидного sellerId (строка)
- **Тип:** Negative
- **Приоритет:** Medium
- **Шаги:**
  1. Отправить GET `/api/1/abc/item`
- **Ожидаемый результат:** HTTP 400 — sellerId должен быть числом

### TC-025: Передача отрицательного sellerId
- **Тип:** Negative
- **Приоритет:** Medium
- **Шаги:**
  1. Отправить GET `/api/1/-1/item`
- **Ожидаемый результат:** HTTP 400 — отрицательный идентификатор продавца невалиден

## GET /api/1/statistic/:id — Статистика объявления

### TC-026: Получение статистики существующего объявления
- **Тип:** Positive
- **Приоритет:** High
- **Предусловия:** Объявление создано со статистикой `likes: 10, viewCount: 50, contacts: 5`
- **Шаги:**
  1. Отправить GET `/api/1/statistic/{id}`
- **Ожидаемый результат:** HTTP 200, значения `likes`, `viewCount`, `contacts` совпадают с переданными при создании

### TC-027: Структура ответа статистики
- **Тип:** Positive
- **Приоритет:** Medium
- **Шаги:**
  1. Создать объявление, запросить статистику
  2. Проверить поля ответа
- **Ожидаемый результат:** Ответ — массив с объектом, содержащим `likes` (int), `viewCount` (int), `contacts` (int)

### TC-028: Статистика объявления с нулевыми значениями

- **Тип:** Corner case
- **Приоритет:** Medium
- **Шаги:**
  1. Создать объявление с `statistics: {likes: 0, viewCount: 0, contacts: 0}`
  2. Запросить статистику
- **Ожидаемый результат:** HTTP 200, все поля статистики равны 0 — нулевые значения валидны для нового объявления

### TC-029: Получение статистики по несуществующему ID
- **Тип:** Negative
- **Приоритет:** High
- **Шаги:**
  1. Отправить GET с несуществующим `id`
- **Ожидаемый результат:** HTTP 404 — статистика по несуществующему объявлению не найдена

### TC-030: Передача невалидного формата ID для статистики
- **Тип:** Negative
- **Приоритет:** Medium
- **Шаги:**
  1. Отправить GET `/api/1/statistic/not-a-uuid`
- **Ожидаемый результат:** HTTP 400 или HTTP 404 (не 500)
